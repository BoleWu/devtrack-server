
一、动态接口传参和controller接收参数问题
1.问题描述：
 #当GET类型接口初始化的时候是这样的http://192.168.1.104:5173/api/dashboard/burn-down，选择内容之后是这样http://192.168.1.104:5173/api/dashboard/burn-down?id=3 后面拼接了一个id，但是这个id是动态的，所以这个接口的参数是动态的;
 #如果controller的方法burndown(Long id)，那么接收的参数无法灵活控制行为，可能无法达到预期的结果;
2.解决方法：
 #使用@RequestParam注解，将参数id绑定到方法参数Long id上，这样参数id就可以动态变化，并且可以灵活控制行为。
推荐使用：@RequestParam(required = false) 这样不管有没有带参数，都会接收到参数，如果是接口没有参数的时候，接收的数据可能是null值，需要判断一下。
3.使用场景：
@GetMapping("/burn-down")
public R<?> burndown(@RequestParam(required = false) Long id){
return R.success(taskService.burndown(id));
}
@Mapper添加if判断是否为空
    @Select("""
    <script>
    SELECT DATE(create_time) day,
    COUNT(*) remain
    FROM task
    WHERE assignee_id = #{userId}
    <if test="id != null and id !=''">
    AND project_id = #{id}
    </if>
    GROUP BY day
    ORDER BY day
    </script>
    """)
    List<BurnDownPointVO> burndown(
    @Param("id") Long id,
    @Param("userId") Long userId
    );
二、Mapper多个参数的问题
1.问题描述：
#使用Mapper接口方法时，接口方法有多个参数，导致执行sql时参数丢失;
2.解决方法：
#Mapper接口方法有多个参数时，那么需要@Param(""),来绑定参数，否则无法接收参数。
3.使用场景：
    @Select("""
    <script>
    SELECT DATE(create_time) day,
    COUNT(*) remain
    FROM task
    WHERE assignee_id = #{userId}
    <if test="id != null and id !=''">
    AND project_id = #{id}
    </if>
    GROUP BY day
    ORDER BY day
    </script>
    """)
    List<BurnDownPointVO> burndown(
    @Param("id") Long id,
    @Param("userId") Long userId
    );
三、@Param 和 @PathVariable  和@Validated的区别
| 注解 | 所属框架 | 作用 | 使用位置 | 典型场景 |
|------|--------|------|--------|--------|
| `@Param` | MyBatis | 给 Mapper 方法参数命名 | Mapper 接口参数 | `#{userId}` 在 SQL 中引用 |
| `@PathVariable` | Spring Web | 提取 URL 路径变量 | Controller 方法参数 | RESTful: `/users/{id}` |
| `@Validated` | Spring Validation | 启用参数校验 | Controller/Service 方法或参数 | 校验 `@RequestBody` 数据合法性 |
